Telco CRM Microservices MVP ER Diagram

Database-per-service yaklasimina gore hazirlanmis Mermaid ER diyagrami. Solid iliskiler servis ici fiziksel FK, dashed iliskiler ise servisler arasi logical external reference anlamina gelir.

Mimari Kapsam

Database-per-service

Her bounded context kendi veritabanina sahiptir: identity_db, customer_db, catalog_db, order_db ve devam eden servis DB'leri.

Internal FK

Ayni servis icindeki tablolar arasinda fiziksel FK kullanilabilir. Diyagramda solid cizgi ile gosterilir.

External reference

Servisler arasi baglar yalnizca ID veya business key saklar. Fiziksel FK degildir; dashed cizgi ile gosterilir.

Eventual consistency

Order, payment, subscription, usage ve billing akislarinda saga, outbox ve event tabanli tutarlilik hedeflenir.

||--o{ gibi iliskiler servis ici sahiplik ve fiziksel FK anlamina gelir.

Dashed relationship

||..o{ gibi iliskiler cross-service logical reference anlamina gelir; DB seviyesinde FK yoktur.

Snapshot fields

Order item ve subscription uzerindeki snapshot JSON alanlari katalog degisikliginden etkilenmeyen tarihsel veriyi korur.

Kaynak servis / alan Hedef servis / entity Not
ORDER_SERVICE_ORDER.customerId CUSTOMER_SERVICE_CUSTOMER.id Logical reference; fiziksel FK yok.
ORDER_SERVICE_ORDER_ITEM.productCode PRODUCT_CATALOG_SERVICE_PRODUCT_OFFERING.code Business key + snapshot; siparis zamani katalog bilgisi kopyalanir.
SUBSCRIPTION_SERVICE_SUBSCRIPTION.customerId CUSTOMER_SERVICE_CUSTOMER.id Musteri birden fazla abonelige sahip olabilir; cross-service FK yok.

BILLING_SERVICE_INVOICE.subscriptionId SUBSCRIPTION_SERVICE_SUBSCRIPTION.id Faturalama servisinde subscription external reference olarak tutulur.
PAYMENT_SERVICE_PAYMENT.invoiceId BILLING_SERVICE_INVOICE.id Odeme, faturaya uygulama seviyesinde baglanir; billing DB'ye FK yok.
TICKET_SERVICE_TICKET.customerId CUSTOMER_SERVICE_CUSTOMER.id Destek kaydi customer master data'yi yalnizca ID uzerinden referanslar.
